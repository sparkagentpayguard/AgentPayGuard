### 1. Reporting Bugs ğŸ›
* Check the [Issues Tab](https://github.com/your-username/AgentPayGuard/issues) to see if the bug has already been reported.
* If not, open a new issue. Please include:
    * A clear and descriptive title.
    * Steps to reproduce the bug.
    * Your environment (OS, Node version, Wallet used, etc.).
    * Expected vs. actual behavior.

### 2. Suggesting Enhancements ğŸ’¡
We love new ideas for Risk Management strategies or Agent Identity features! 
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful to the AgentPayGuard ecosystem.

### 3. Pull Requests (PRs) ğŸ› ï¸
This is the most direct way to contribute. Follow these steps:
1. **Fork** the repository.
2. **Clone** your fork: `git clone https://github.com/your-username/AgentPayGuard.git`.
3. **Create a branch**: `git checkout -b feature/amazing-new-strategy`.
4. **Make your changes**. Please ensure your code follows our style guides.
5. **Commit your changes**: Use descriptive commit messages (e.g., `feat: add EIP-3009 gasless support`).
6. **Push to the branch**: `git push origin feature/amazing-new-strategy`.
7. **Open a Pull Request** against our `main` branch.

## ğŸ“ Style Guide

### Code Style

* **TypeScript:** We use Prettier and ESLint. Please run `pnpm lint` before committing.
* **Solidity:** Follow the [Solidity Style Guide](https://docs.soliditylang.org/en/latest/style-guide.html). Ensure all contracts are documented with NatSpec.

### Commit Messages

We follow [Conventional Commits](https://www.conventionalcommits.org/):

* `feat:` for new features.
* `fix:` for bug fixes.
* `docs:` for documentation changes.
* `refactor:` for code changes that neither fix a bug nor add a feature.
