# AgentPayGuard 演示视频制作指南（前端演示版）

> **目标**: 制作一个5-8分钟的演示视频，通过前端界面展示 AgentPayGuard 的核心功能和创新点

---

## 视频结构建议

### 第一部分：问题陈述与系统概览 (30秒)

**画面**:
- 打开前端首页 (http://localhost:5173)
- 展示系统状态面板（ONLINE、SECURITY MAXIMUM、NETWORK、2/3 MULTISIG）
- 展示能力卡片（PAY、AI CHAT、MULTI-SIG、FREEZE、PROPOSALS）

**讲稿**:
> "AI Agent 一旦开始'真花钱'，系统立刻出现三个现实问题：谁在花钱？能不能停得住？事后能不能说清？AgentPayGuard 是在 Kite 上的一次具体实现，我们围绕这三个问题构建了完整的解决方案。现在让我通过前端界面为大家展示。"

**英文版本**:
> "When AI Agents start making real payments, three critical questions emerge: Who is spending? Can we stop it when needed? Can we explain it afterward? AgentPayGuard is a concrete implementation on Kite that addresses all three questions. Let me demonstrate through our web interface."

---

### 第二部分：核心功能演示 (4-5分钟)

#### 2.1 自然语言支付 - AI Chat (1.5分钟)

**演示步骤**:
1. 点击首页的 "AI CHAT" 卡片，进入 AI Chat 页面
2. 连接钱包（MetaMask），切换到 Kite Testnet
3. 在聊天框输入：`"Pay 50 USDC to 0xd2d45ef2f2ddaffc8c8bc03cedc4f55fb9e97e2b for server hosting"`
4. 展示 AI 解析结果卡片（收款地址、金额、币种、目的、置信度）
5. 展示 AI 风险评估卡片（风险分数、等级、理由、建议）
6. 展示策略检查结果（通过/拒绝）
7. （可选）展示链上交易结果（交易哈希、区块浏览器链接）

**讲稿**:
> "首先，我们支持自然语言支付。在 AI Chat 页面，Agent 可以接受如 'Pay 50 USDC to 0x... for server hosting' 的指令。系统会自动解析收款地址、金额、币种和支付目的，并显示置信度。然后进行 AI 风险评估，我们可以看到风险分数、等级和详细的风险理由。通过策略检查后，支付将在链上执行。整个过程都在前端界面实时展示，用户可以清楚地看到每一步的决策过程。"

**关键点**:
- 强调自然语言理解能力
- 展示 AI 风险评估的透明性
- 说明策略检查的严格性
- 展示实时进度指示器

---

#### 2.2 多层安全防护 - Dashboard & Freeze (1.5分钟)

**演示步骤**:
1. 进入 Dashboard 页面，展示策略配置面板
   - 展示白名单地址列表
   - 展示单笔限额（MAX_AMOUNT）
   - 展示日限额（DAILY_LIMIT）
2. 返回 AI Chat，尝试向非白名单地址支付
   - 展示被拒绝的结果（NOT_IN_ALLOWLIST）
3. 尝试超过限额的支付
   - 展示被拒绝的结果（AMOUNT_EXCEEDED）
4. 进入 Freeze 页面
   - 输入一个地址，点击 "SCAN" 查询冻结状态
   - 展示 "ACTIVE" 或 "FROZEN" 状态
5. 如果当前钱包是多签 owner，展示提交冻结提案的流程
   - 点击 "SUBMIT FREEZE PROPOSAL"
   - 展示提案已提交（需要第二个 owner 确认）

**讲稿**:
> "AgentPayGuard 采用多层安全防护。在 Dashboard 页面，我们可以看到策略配置：白名单地址列表、单笔限额和日限额。如果收款地址不在白名单中，支付会被立即拒绝。如果金额超过限额，同样会被拒绝。在 Freeze 页面，我们可以查询任何地址的链上冻结状态。如果地址被多签冻结，Agent 无法向其转账。这确保了即使在 AI 评估通过的情况下，人类仍可以通过多签机制紧急叫停。"

**关键点**:
- 强调"执行前检查"而非"事后追责"
- 展示规则引擎的严格性
- 说明多签冻结的紧急响应能力
- 展示链上状态查询

---

#### 2.3 多签治理 - Proposals (1分钟)

**演示步骤**:
1. 进入 Proposals 页面
2. 展示提案列表（如果有）
   - 展示提案 ID、类型（freeze/unfreeze）、目标地址
   - 展示确认进度（1/2、2/2）
   - 展示执行状态（pending/executed）
3. 如果当前钱包是多签 owner：
   - 点击 "NEW" 按钮创建新提案
   - 选择类型（freeze/unfreeze）
   - 输入目标地址
   - 提交提案（自动确认一次）
4. 切换到另一个 owner 钱包
   - 确认提案（达到 2/2 阈值）
   - 执行提案
5. 返回 Freeze 页面验证冻结状态已更新

**讲稿**:
> "多签治理是 AgentPayGuard 的核心安全机制。在 Proposals 页面，我们可以看到所有冻结和解冻提案。每个提案需要 2/3 的 owner 确认才能执行。这里我们可以看到提案的确认进度和执行状态。作为多签 owner，我可以提交新提案、确认其他 owner 的提案，并在达到阈值后执行。这确保了所有关键操作都需要多数人同意，防止单点故障。"

**关键点**:
- 展示多签治理流程
- 说明 2/3 阈值的安全性
- 展示提案的完整生命周期

---

#### 2.4 传统支付与系统监控 (1分钟)

**演示步骤**:
1. 进入 Pay 页面
   - 展示支付表单（收款地址、金额、支付模式 EOA/AA）
   - 选择 "Execute real on-chain transaction"
   - 提交支付请求
   - 展示支付结果（成功/失败）
2. 返回 Dashboard
   - 展示多签钱包信息（合约地址、阈值、owners）
   - 展示策略配置（白名单数量、限额）
   - 展示统计信息（今日交易数、总提案数）
3. 进入 History 页面（如果有）
   - 展示历史交易记录

**讲稿**:
> "除了自然语言支付，我们也支持传统的支付方式。在 Pay 页面，用户可以手动输入收款地址和金额，选择 EOA 或 AA 模式。在 Dashboard 页面，我们可以看到系统的完整状态：多签钱包信息、策略配置和统计信息。所有操作都通过 API 与后端交互，确保了安全性和一致性。"

**关键点**:
- 展示传统支付方式
- 说明系统监控能力
- 展示前后端分离架构

## 录制技巧

### 准备工作

1. **环境准备**:
   - 启动后端服务器：`pnpm server`（确保 API 正常运行）
   - 启动前端：`cd frontend && npm run dev`（访问 http://localhost:5173）
   - 准备至少 2 个测试钱包（用于多签演示）
   - 确保钱包有足够的 KITE 测试币
   - 配置好 `.env` 文件（后端配置）

2. **前端准备**:
   - 确保前端已构建并可以正常运行
   - 准备测试数据（白名单地址、测试收款地址）
   - 确保 MetaMask 已连接到 Kite Testnet
   - 准备多个测试场景（正常支付、被拒绝、高风险等）

3. **录制工具**:
   - 屏幕录制：OBS Studio、QuickTime（Mac）、Windows 录屏
   - 音频：使用外接麦克风，确保音质清晰
   - 编辑：剪映、Final Cut Pro、Premiere Pro
   - 浏览器：Chrome/Edge（推荐），使用无痕模式避免干扰

4. **演示数据准备**:
   - 准备多个测试场景（正常支付、被拒绝、高风险等）
   - 准备链上交易哈希链接（用于展示）
   - 准备前端界面的关键截图（备用）

### 录制建议

1. **画面清晰度**:
   - 使用高分辨率录制（至少 1920x1080）
   - 浏览器缩放设置为 100%（确保界面清晰）
   - 使用深色主题（前端已默认深色主题）
   - 确保前端界面完全可见，不要遮挡关键信息

2. **节奏控制**:
   - 每个页面切换之间停顿 1-2 秒
   - 关键操作（如提交支付、确认提案）可以慢速展示
   - 等待 API 响应时，可以暂停讲解，等待结果出现
   - 避免过快切换页面

3. **交互演示**:
   - 鼠标移动要流畅，不要过快
   - 点击按钮时稍作停顿，确保观众看清
   - 展示表单输入时，可以逐字输入或快速输入后暂停
   - 展示结果时，用鼠标高亮关键信息

4. **错误处理**:
   - 如果演示中出现错误（如网络错误、API 错误），可以暂停录制
   - 展示错误提示信息，说明这是正常的错误处理机制
   - 或者准备备用测试数据

5. **后期编辑**:
   - 添加标题和转场效果
   - 添加关键信息标注（如"AI 风险评估"、"策略检查"、"多签确认"）
   - 在重要操作时添加放大效果（如提交按钮点击）
   - 添加背景音乐（可选，音量要低）
   - 可以添加箭头或高亮框标注关键信息

## 前端演示详细步骤

### 步骤1: 启动服务

```bash
# 终端1: 启动后端
pnpm server

# 终端2: 启动前端
cd frontend
npm run dev
```

访问：http://localhost:5173

### 步骤2: 连接钱包

1. 打开前端首页
2. 点击右上角 "CONNECT" 按钮
3. 选择 MetaMask
4. 切换到 Kite Testnet（Chain ID: 2368）
5. 确认连接

### 步骤3: AI Chat 演示

1. 点击首页 "AI CHAT" 卡片
2. 输入自然语言支付指令
3. 观察解析结果、风险评估、策略检查
4. 确认支付（如果通过）

### 步骤4: Dashboard 演示

1. 点击首页 "MULTI-SIG" 卡片或导航栏 "DASHBOARD"
2. 查看策略配置（白名单、限额）
3. 查看多签钱包信息
4. 查看统计信息

### 步骤5: Freeze 演示

1. 点击首页 "FREEZE" 卡片
2. 输入地址，点击 "SCAN"
3. 查看冻结状态
4. 如果是 owner，可以提交冻结/解冻提案

### 步骤6: Proposals 演示

1. 点击首页 "PROPOSALS" 卡片或导航栏
2. 查看提案列表
3. 如果是 owner，可以创建、确认、执行提案
4. 切换钱包演示多签确认流程

### 步骤7: Pay 演示

1. 点击首页 "PAY" 卡片
2. 输入收款地址和金额
3. 选择支付模式（EOA/AA）
4. 提交支付
5. 查看结果
