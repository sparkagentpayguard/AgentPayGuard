Kite AI 支付赛道｜技术使用规则
（Payment Track）
一、赛道目标
本赛道鼓励参赛者基于 Kite AI 技术文档（docs.gokiteai.com），构建可运行的 AI Agent 支付应用场景，实现：
●AI 智能体可在链上完成真实或测试网稳定币支付
●展示自动化结算与基础风控能力
适合新人开发者与首次接触 AI + Web3 技术的参赛者参与。

二、技术使用范围
参赛项目需至少使用 Kite AI 提供的以下核心能力之一：
●Agent 身份系统（Agent / Passport）
●稳定币支付接口（Stablecoin Payment）
●支付权限 / 额度控制
●官方 SDK / API / 合约组件
📌 不限制具体语言与框架，但需基于官方文档完成集成。

三、新人入门最低实现要求（获奖门槛）
参赛作品必须同时满足以下 4 项基础要求，否则不进入评奖阶段：
项目	要求
链上支付	至少完成 1 笔真实或测试网稳定币转账
Agent 身份	使用 Kite Agent 或身份体系
权限控制	设置基础支付额度 / 规则
可复现性	提供完整运行说明或演示
👉 简单理解：能跑通「创建 Agent → 发起支付 → 成功到账」完整流程。

四、作品提交规范
参赛队伍需提交以下材料：
1️⃣ 项目代码仓库（GitHub / GitLab）
2️⃣ 项目说明文档（README）包含但不限于：
●环境搭建方法
●运行步骤
●核心功能说明
3️⃣ 演示视频或 Demo 链接（可选但加分）
