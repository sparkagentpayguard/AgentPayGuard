# AgentPayGuard 演示视频制作指南

> **目标**: 制作一个5-8分钟的演示视频，展示 AgentPayGuard 的核心功能和创新点

---

## 视频结构建议

### 第一部分：问题陈述 (30秒)

**画面**:
- 简洁的标题画面："AgentPayGuard - AI Agent 支付风控系统"
- 问题痛点可视化（可选）

**讲稿**:
> "AI Agent 一旦开始'真花钱'，系统立刻出现三个现实问题：谁在花钱？能不能停得住？事后能不能说清？AgentPayGuard 是在 Kite 上的一次具体实现，我们围绕这三个问题构建了完整的解决方案。"

**英文版本**:
> "When AI Agents start making real payments, three critical questions emerge: Who is spending? Can we stop it when needed? Can we explain it afterward? AgentPayGuard is a concrete implementation on Kite that addresses all three questions."

---

### 第二部分：核心功能演示 (4-5分钟)

#### 2.1 自然语言支付 (1.5分钟)

**演示步骤**:
1. 打开终端，运行 `pnpm demo:ai-agent "Pay 50 USDC to 0xd2d45ef2f2ddaffc8c8bc03cedc4f55fb9e97e2b for server hosting"`
2. 展示 AI 解析结果（收款地址、金额、币种、目的）
3. 展示 AI 风险评估（分数、等级、理由）
4. 展示策略检查通过
5. （可选）展示链上交易结果

**讲稿**:
> "首先，我们支持自然语言支付。Agent 可以接受如 'Pay 50 USDC to 0x... for server hosting' 的指令。系统会自动解析收款地址、金额、币种和支付目的，然后进行 AI 风险评估。这里我们可以看到，AI 给出了风险分数 25，等级为低风险，并提供了详细的风险理由。通过策略检查后，支付将在链上执行。"

**关键点**:
- 强调自然语言理解能力
- 展示 AI 风险评估的透明性
- 说明策略检查的严格性

---

#### 2.2 多层安全防护 (1.5分钟)

**演示步骤**:
1. 展示策略配置（白名单、限额）
2. 演示非白名单地址被拒绝
3. 演示超过限额被拒绝
4. 展示链上冻结检查
5. （可选）展示多签冻结操作

**讲稿**:
> "AgentPayGuard 采用多层安全防护。首先是传统规则：白名单、单笔限额、日限额。如果收款地址不在白名单中，支付会被立即拒绝。如果金额超过限额，同样会被拒绝。此外，每次支付前都会检查链上冻结状态。如果地址被多签冻结，Agent 无法向其转账。这确保了即使在 AI 评估通过的情况下，人类仍可以通过多签机制紧急叫停。"

**关键点**:
- 强调"执行前检查"而非"事后追责"
- 展示规则引擎的严格性
- 说明多签冻结的紧急响应能力

---

#### 2.3 Agent 身份绑定 (1分钟)

**演示步骤**:
1. 展示 Agent 身份初始化日志
2. 展示支付请求与 Agent 身份绑定
3. 展示链上交易中的 Agent 身份信息

**讲稿**:
> "每个支付请求都与 Agent 身份绑定。我们支持两种方式：KitePass API Key 或 Kite AA SDK 账户抽象。这确保了每笔支付都可以追溯到具体的 Agent，满足了'谁在花钱'的问题。在链上，每笔交易都包含 Agent 身份信息，形成了完整的审计轨迹。"

**关键点**:
- 强调身份可验证性
- 说明审计轨迹的完整性

---

#### 2.4 前端界面演示 (1分钟，可选)

**演示步骤**:
1. 打开前端界面
2. 展示支付页面（Pay）
3. 展示 AI Pay 页面（自然语言支付）
4. 展示 AI Chat 页面（AI 对话式支付）
5. 展示 Dashboard（策略配置、性能监控、历史记录）
6. 展示 Freeze 页面（冻结状态、多签提案）
7. （可选）展示 Metrics Dashboard（实时性能指标）

**讲稿**:
> "我们还提供了完整的前端界面。用户可以通过 Web UI 发起支付、使用自然语言与 AI 对话、查看策略配置、查看性能监控和历史记录，以及管理多签冻结。所有操作都通过 API 与后端交互，确保了安全性和一致性。"

**关键点**:
- 展示用户体验
- 说明前后端分离架构

---

### 第三部分：技术亮点 (1-2分钟)

**讲稿**:
> "AgentPayGuard 的技术亮点包括：第一，模型无关的流水线设计，未来可以无缝切换到本地或边缘部署的大模型，实现亚秒级响应。第二，多层安全防护，结合规则引擎、AI 风险评估、ML 风险检测（MVP 版本）和链上冻结检查。第三，完整的审计轨迹，每笔支付都可以在链上查验。第四，可扩展的 API 设计，其他 Dapp 可以复用我们的策略和风控层。"

**关键点**:
- 强调技术架构的前瞻性
- 说明安全设计的深度
- 展示可扩展性

---

### 第四部分：技术深度与未来改进 (30秒)

**讲稿**:
> "AgentPayGuard 已经实现了基于机器学习的风险检测模块，包括 59 维特征工程、XGBoost 风险预测模型和异常检测。当前实现是简化 MVP 版本，用于演示和概念验证。未来，我们将支持本地模型部署，实现亚秒级响应。长期来看，我们将探索可验证推理和零知识可验证 Agent 凭证，与 Kite 白皮书的未来方向保持一致。"

**关键点**:
- 说明 ML 模块已实现（MVP 版本）
- 展示技术路线图
- 说明与 Kite 生态的契合度

## 录制技巧

### 准备工作

1. **环境准备**:
   - 确保所有演示脚本可以正常运行
   - 准备测试钱包和测试币
   - 配置好 `.env` 文件

2. **录制工具**:
   - 屏幕录制：OBS Studio、QuickTime（Mac）、Windows 录屏
   - 音频：使用外接麦克风，确保音质清晰
   - 编辑：剪映、Final Cut Pro、Premiere Pro

3. **演示数据准备**:
   - 准备多个测试场景（正常支付、被拒绝、高风险等）
   - 准备链上交易哈希链接
   - 准备前端界面截图

### 录制建议

1. **画面清晰度**:
   - 使用高分辨率录制（至少 1920x1080）
   - 终端字体大小适中，确保代码可读
   - 使用深色主题，减少视觉疲劳

2. **节奏控制**:
   - 每个演示步骤之间停顿 1-2 秒
   - 关键操作可以慢速展示
   - 避免过快切换画面

3. **错误处理**:
   - 如果演示中出现错误，可以暂停录制，修复后继续
   - 或者准备备用演示脚本

4. **后期编辑**:
   - 添加标题和转场效果
   - 添加关键信息标注（如"AI 风险评估"、"策略检查"）
   - 添加背景音乐（可选，音量要低）
